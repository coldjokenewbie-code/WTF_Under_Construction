## 查證結果（claude-code-guide，官方文件 code.claude.com/docs）

1. **PreToolUse hook 可設在 user 層級 `~/.claude/settings.json`，全專案全域生效**——前一 TaskLog 記載 B 方案因「只能到專案層級」不採用，此前提與官方文件不符。deny 理由會回饋模型使其修正（hooks-guide：「Claude receives it as feedback so it can adjust」）；matcher 可比對 `tool_input.file_path`。
2. **Stop hook** 回 `{"ok": false, "reason": ...}` 可擋停並強制模型繼續修；防無限迴圈機制文件未載明。
3. **CLAUDE.md／常載規則官方建議 <200 行**（memory.md：「Longer files consume more context and reduce adherence」）——現行 hook 注入 63,779 字元反而稀釋遵循度，方向應為濃縮。
4. subagent `.claude/agents/*.md` 內文＝該 agent 的 system prompt（規則可進最高層級）。
5. （未驗證）`--append-system-prompt` 於 interactive 模式的支援度，文件主要寫在 headless 節。
