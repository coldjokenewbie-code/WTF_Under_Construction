共用概念

三工具 headless CLI 一覽

工具headless 指令stdin 陷阱
Claude claude -p "<prompt>" 直接讀寫檔案會卡住；prompt 只要求輸出文字
Antigravity agy -p "<prompt>" 或 agy --print "<prompt>" 寫檔若觸發權限確認會卡；加 --dangerously-skip-permissions 可跳過
Codex codex exec "<prompt>" < /dev/null 缺少 < /dev/null 會等待 stdin，直接卡死

基本原則

  • 三者 headless 皆單次無狀態（每次新 session）；多輪討論由 Tech Lead 每次把脈絡餵進 prompt。
  • 被呼叫的 agent 只輸出文字，不讀寫檔；Tech Lead 統一落檔，避免各自寫檔引發衝突或亂碼。
  • 卡住超過 90 秒：砍程序，縮短 prompt，重跑。不等也不跳過——追原因（常見：讀寫檔指令、stdin 阻塞、權限確認彈窗）。

Signal 格式

事件格式
討論完成DISCUSSION|<AgentID>|<相對路徑>|YYYY-MM-DD
任務完成DONE|<AgentID>|<相對路徑>|<Timestamp>

寫入位置：<專案根>/AGENT_SIGNAL.log（非 _context/）。

呼叫 Claude

  • 指令：claude -p "$(cat /tmp/claude_prompt.txt)"
  • Codex 沙盒內若回 Not logged in：請使用者在 Terminal 執行 claude /login，再讓 Codex 以「沙盒外核准」方式執行一次即可。token 本身不需重新登入，問題是沙盒執行權限。
  • 禁止在 prompt 要求 Claude 讀取或寫入檔案——會直接卡住。背景文字貼進 prompt，Claude 只輸出。

呼叫 Antigravity

  • 指令：agy --print "$(cat /tmp/agy_prompt.txt)"
  • 可讓 Antigravity 直接 append 指定檔案，但 prompt 內明確限制 scope（只改這兩個檔、不改其他）。
  • 收到 signal 後先清理編碼，再整理內容（見下方腳本）。

編碼風險：Antigravity append 中文檔案時曾產生非 UTF-8 位元組。收到回覆後先跑清理腳本，不要直接沿用原稿。

# 清理非 UTF-8 位元組
python3 -c "
with open('_context/討論檔.md','rb') as f: raw=f.read()
with open('_context/討論檔.md','w',encoding='utf-8') as f: f.write(raw.decode('utf-8',errors='replace'))
"

1. 讀討論檔：<ABS_PROJECT_PATH>/_context/AI_TEAM_DISCUSSION_YYYY-MM-DD_topic.md

呼叫 Claude

  • 指令：claude -p "<prompt>" < /dev/null
  • < /dev/null 防止 stdin 阻塞。Claude headless 模式不需互動輸入，但某些版本會等待——加上去保險。
  • 同樣禁止在 prompt 要求 Claude 讀寫檔，只要求輸出文字。

呼叫 Codex

  • 指令：codex exec "<prompt>" < /dev/null 或縮寫 codex e "<prompt>" < /dev/null
  • < /dev/null 是必要的：缺少時 Codex 偵測到互動式 stdin 會等待輸入，卡死在 Reading additional input from stdin...。
  • 同樣要求只輸出文字，由 Antigravity（Tech Lead）攔截輸出後統一落檔。

Prompt 範本：呼叫 Claude / Codex

請用繁體中文台灣用語，針對 <任務主題> 給 Tech Lead（Antigravity）意見。

背景：
- <環境>
- <資料來源>

請只輸出四段：
## Recommendation
## Detailed Steps
## Risks
## Validation

不要讀寫檔。輸出純文字即可，Tech Lead 統一落檔。

卡住處理

症狀處理
Claude 卡死確認有加 < /dev/null；縮短 prompt；確認無讀寫檔指令。
Codex 卡在 Reading additional input...必定是缺少 < /dev/null，補上重跑。
Codex 回應後未結束加 -s read-only 旗標限制寫入，或改用 --skip-git-repo-check。
