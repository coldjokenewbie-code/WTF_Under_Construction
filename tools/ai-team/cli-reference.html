用途：記錄由 Codex 擔任 Tech Lead，呼叫 Antigravity 與 Claude 補充意見的完整流程、提示詞、腳本與各 agent 正式規格，系統化為 WTF 共用 ai-team CLI 協作手冊。

本次成功模式

使用者只跟 Codex 溝通；Codex 透過 CLI 呼叫其他 agent。

Antigravity 可直接讀指定檔案並 append 討論檔。Claude 直接讀寫檔案時曾卡住，改成只要求輸出文字意見，再由 Codex 寫入檔案，較穩定。

Agent 呼叫方式 本次狀態
Antigravity agy --print "<prompt>" 成功讀檔、append 討論檔、寫入 AGENT_SIGNAL.log；曾產生非 UTF-8 亂碼，需由 Codex 清理。
Claude claude -p "<prompt>" 使用者 Terminal 已登入後，Codex 需用沙盒外核准執行。直接讀寫檔案會卡住；短 prompt 只輸出文字可成功。

建議流程

  1. Codex 先建立討論檔，例如 _context/AI_TEAM_DISCUSSION_YYYY-MM-DD_topic.md。
  2. 討論檔放完整背景、資料路徑、問題、回覆格式與 Codex 初始立場。
  3. 先呼叫 Antigravity，scope 限定只讀指定檔案、只 append 指定討論檔與 signal。
  4. 呼叫 Claude 時，優先使用「只輸出文字，不讀寫檔」模式。
  5. Codex 讀取各 agent 回覆後，親自整理進討論檔與最終文件。
  6. 寫入 AGENT_SIGNAL.log 作為流程紀錄。
  7. 若 agent 卡住超過 90 秒，查程序並停止，改用短 prompt 或檔案中繼。

Antigravity Prompt 範本

你是 Antigravity，被納入一個 ai-team 協作。Tech Lead 是 Codex，它已開了一份討論檔要你回覆。

請依序：
1. 讀討論檔：<ABS_PROJECT_PATH>/_context/AI_TEAM_DISCUSSION_YYYY-MM-DD_topic.md
2. 讀交接文件補背景：<ABS_PROJECT_PATH>/_context/<handover-file>.md
3. 在討論檔末尾 append 你的回覆段，標題格式：## Antigravity Reply — YYYY-MM-DD
   內容用以下結構（繁體中文台灣用語）：
   ### Recommendation
   ### Detailed Steps
   ### Formulas / Properties
   ### Risks
   ### Validation
4. 完成後在 <ABS_PROJECT_PATH>/AGENT_SIGNAL.log append 一行：
   DISCUSSION|Antigravity|_context/AI_TEAM_DISCUSSION_YYYY-MM-DD_topic.md|YYYY-MM-DD

只做這些，不要改其他檔案。完成後回報你寫了什麼。

Antigravity 呼叫腳本

cat > /tmp/agy_prompt.txt << 'EOF'
<貼上 Antigravity Prompt>
EOF

agy --print "$(cat /tmp/agy_prompt.txt)"

卡住處理

症狀 處理
claude -p 回 Not logged in 請使用者在 Terminal 執行 claude /login，回 shell 後再測 claude -p "請只回覆 OK"。
使用者 Terminal 可用，但 Codex 仍回未登入 改用沙盒外核准執行 claude -p。本次實測可成功。
Claude 讀寫檔案 prompt 卡住 停止程序，改用「不要讀寫檔，只輸出文字」短 prompt。
討論檔出現非 UTF-8 亂碼 由 Codex 清理無效位元組後再整理內容。最終文件不要直接使用外部 agent append 的原稿。

Claude CLI 正式規格

補充者：[Claude@comaMacBookAir] 2026-06-08

登入

  1. 在使用者 Terminal（非 Codex 沙盒）執行：claude /login，瀏覽器完成 OAuth。
  2. 登入後驗證：claude -p "請只回覆 OK"，若回 OK 即成功。
  3. Codex 沙盒若仍回 Not logged in：讓使用者「允許（沙盒外核准）」該指令執行一次，Codex 即可繼續呼叫。token 本身不需要重新登入；問題是沙盒執行權限，不是認證失效。

短 prompt 原則

  • 使用 claude -p "<prompt>"（非互動，單一輸出後結束）。
  • 背景資料直接貼進 prompt 文字，不靠 Claude 自行讀取檔案。
  • prompt 結尾明確標注輸出格式（## Recommendation / ## Risks 等），Tech Lead 直接複製。
  • prompt 過長（> 2000 字）時，先存 /tmp/claude_prompt.txt，用 claude -p "$(cat /tmp/claude_prompt.txt)" 呼叫，避免 shell 轉義斷行。

避免直接讀寫檔

已知問題：要求 Claude 在 CLI 模式下讀取或 append 檔案，會因等待 stdin/stdout 確認而卡住，90 秒後 timeout。

  • 禁止在 prompt 內加入「讀 _context/xxx.md」「append 到 討論檔」等指令。
  • 正確模式：背景文字貼進 prompt → Claude 只輸出結構化文字 → Tech Lead（Codex）寫入檔案。
  • 若背景檔案很大：cat <file> 管入 prompt 字串（見 prompt 範本）。Tech Lead 自行控制截斷長度。

卡住處理（Claude 專項）

症狀處理
Not logged in 使用者 Terminal 執行 claude /login，再用沙盒外核准讓 Codex 呼叫。
90 秒無輸出 Ctrl+C，確認 prompt 是否含讀寫檔指令，改用純輸出模式再試。
輸出中斷（截半） prompt 末尾加 請分段回覆，每段不超過 400 字，或改用 /tmp 檔案中繼。
反覆失敗 改用最短 prompt 驗證（claude -p "請只回覆 OK"），確認工具本身是否正常，再逐步加回背景。

Antigravity CLI 正式規格

整理者：[Codex@Mac] 2026-06-08（待 Antigravity 本人補充後合併）

權限需求

  • 讀取：專案目錄下所有 _context/ 檔案（使用絕對路徑，相對路徑易因工作目錄不同而失敗）。
  • 寫入：只需 _context/討論檔.md 與 AGENT_SIGNAL.log 兩個檔案，其餘目錄不需寫入。
  • 呼叫前確認 agy 對專案目錄有讀寫權；macOS 需確認 Terminal 的「完整磁碟存取」或已給予資料夾存取授權。

寫檔限制

  • prompt 內明確限制 scope：「只讀這兩個檔，只改這兩個檔」，否則 Antigravity 可能主動整理或新增其他檔案。
  • prompt 末尾加：只做這些，不要改其他檔案。完成後回報你寫了什麼。
  • Tech Lead 收到 DONE signal 後，必須親自 diff 或 git status 確認實際修改範圍，不信任 Antigravity 的口頭回報。

編碼風險

已知問題：Antigravity append 中文討論檔時，曾產生非 UTF-8 位元組，導致後續工具讀取失敗或顯示亂碼。

  • Codex 收到 signal 後，先執行編碼清理再整理內容：
# 清理非 UTF-8 位元組（保留可讀內容，丟棄無效字元）
python3 -c "
with open('_context/討論檔.md', 'rb') as f: raw = f.read()
clean = raw.decode('utf-8', errors='replace')
with open('_context/討論檔.md', 'w', encoding='utf-8') as f: f.write(clean)
print('清理完成')
"
  • 最終文件（outputs HTML、TaskLog）不直接使用 Antigravity 的 append 原稿；Tech Lead 重新撰寫，只引用意見內容。

Signal 寫入慣例

  • 格式（一行）：DISCUSSION|Antigravity|<相對於專案根的路徑>|YYYY-MM-DD
  • 寫入檔案：<ABS_PROJECT_PATH>/AGENT_SIGNAL.log（專案根，非 _context/）。
  • Tech Lead 監聽此 log 判斷任務完成，不要等 Antigravity 在聊天視窗口頭回報。
