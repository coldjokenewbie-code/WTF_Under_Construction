# Dev Workflow (開發節奏)

## 階段一：規劃 (Planning)
- **禁止直接改碼**：先釐清目標與根本原因。
- **產出實作計畫**：修改前必須產出 `implementation_plan.md`，列出預計更動的檔案與架構變化。
- **使用者確認**：計畫確認無誤後，才進入執行階段。

## 階段二：執行 (Execution)
- **拆解任務**：建立 `task.md` (Checklist)，將大任務拆為小步驟。
- **單步執行**：每次只專注完成 Checklist 上的一個步驟，完成後更新狀態，避免越改越亂。

## 階段三：驗證與總結 (Verification)
- **測試**：每完成一段功能，必須進行預期行為驗證（靜態分析或實際執行測試）。
- **總結**：產出 `walkthrough.md` 或簡短總結，記錄改動內容與測試結果，再進入下一環節。
