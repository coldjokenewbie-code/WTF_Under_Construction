# 展場互動骨架：切分裁決書
> Fable 5 依 6+ 展場案的 lessons 與交付物裁決「什麼進骨架、什麼是選項、什麼是特例」。2026-07-06。
> 小隊依 `extraction-process.md` 七步把裁決落成 starter kit 時，以本檔為第 3-4 步的既定答案；發現與實例矛盾→標「⚠️待裁決」，不得自行推翻。

## 骨架（每個新案都要,不問）
| 項 | 內容 | 依據 |
|---|---|---|
| 待機迴圈 | idle 計時→attract 頁；**離開每個狀態的路徑都要清計時器** | 國圖南 outroTimer 事故 |
| 右鍵封鎖 | kiosk 一律 `contextmenu preventDefault` | 手勢與選單互吞事故 |
| 誤觸防護 | 手勢中開啟覆蓋層→一次性旗標消費下一個 pointerup（禁時間窗） | 慢速拖曳超時事故 |
| 離線資產 | 零 CDN：字體/圖/腳本全本地（`document.fonts.check` 斷言） | 展機離線需求＋networkidle 卡死 |
| 自驗腳本 | 每案標配 Playwright 煙霧測試：pageerror/console 監聽＋逐互動斷言＋多輪連續（單輪測不出計時器類 bug） | 「畫面對但互動壞」多次 |
| 防火牆檢查表 | 區網部署必含 Public profile 入站規則＋平板端連通驗證 | CDIC kiosk 靜默斷線 |
| 視覺驗收 | 只認算繪截圖 vs 錨點（`anchors/README.md` 鐵律） | 80/10 灌水事故 |
| 出處標註 | 史料影像必附來源行 | 標案基本動作，見錨-P1 |

## 選項（依意圖選,附選用條件）
| 項 | 條件 |
|---|---|
| 版面骨架：滿版氛圍型 vs 資訊分層型 | 錨-P3：這一頁的工作是情緒還是操作 |
| 觸控接管閾值（dx>14 且 dx>1.3dy 等） | 有頁內捲動與橫滑並存才需要 |
| baked 文字補丁法 | 只在不能重製原圖時用 |
| 曲線導引版面（DOM 實位繪 path） | 內容長度不定且要路徑敘事才用；成本高 |

## 特例（禁入骨架）
- 各案的業主色彩/字型規範、RFID/硬體介接、特定展品資料結構。
- 任何「只在一個案子出現且原因是該案需求」的做法——出現在骨架裡即為第 6 步該抓的過度抽象。

## 待萃取（小隊執行,七步流程）
從 cowork_CDIC、SouthLibrary、竹科教館、組裝互動四案實碼提取上表骨架項的**最佳實作版本**成 `kit/`（HTML 模板＋手勢層 js＋verify 腳本模板），試用驗證用假新案跑一輪。
