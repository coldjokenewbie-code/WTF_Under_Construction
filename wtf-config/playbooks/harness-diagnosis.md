# 快速診斷：本 harness 三大漏洞與修法
> 適用：所有 Claude 工具（Code／Cowork／Chat）與其他 AI agent
> 來源：Fable 5 session 2026-07-03 盤點。證據引用備份版 `wtf-config/archive/2026-07-03_pre-fable5/`（行號以備份為準，現行檔已照本診斷改寫）
> 讀法：其他制度檔（model-dispatch、judgment-rubrics）引用本檔；平時不必常載，出現「context 不夠／答非所問／說完成但沒完成」時回來對照

## 第 1 名：開場全量載入（最漏 token）

**病灶**：舊規則要求開場「讀取 `_context/` 中所有 `.md` 檔案」（archive GLOBAL.md:31、CLAUDE_CODE.md:10），同一檔又說「進場先讀 INDEX，不必掃全部 `_context/`」（archive GLOBAL.md:183）。兩條矛盾。弱模型遇到矛盾規則會選字面較明確的那條＝全讀。`_context/` 現有 12+ 個進行中檔案、數千行，每個 session 開場先燒掉一大塊 context，之後長對話更早觸發摘要壓縮、更早遺忘開頭的驗收條件。

**修法（照做即可）**：開場只讀三檔，其餘按 INDEX 指路——
1. `_context/INDEX.md`（現況與指路）
2. INDEX 指到的「當前 TaskLog」一份（todo 真相源）
3. `_context/lessons-learned.md`（教訓，永遠讀）

其他 `_context/` 檔案：**只在 INDEX 的「讀取指引」點名、或使用者點名時才讀**。`rules/` 照舊全讀（通常很短）。此修法已寫入現行 GLOBAL.md／CLAUDE_CODE.md，若發現任何檔案又出現「讀取所有 .md」字樣，視為退化，改回三檔制。

**判準**：開場載入（不含系統注入）超過約 800 行文字＝有問題，回頭檢查是不是又全讀了。

## 第 2 名：主對話自己下場做粗活（最易失焦）

**病灶**：主對話直接掃 repo、整檔閱讀長檔、把工具原始輸出留在對話裡。context 被低價值內容填滿 → 觸發自動摘要 → 使用者最初的指令、驗收條件、中途決策被壓縮遺失 → 後半段答非所問或重做已做過的事。這不是模型能力問題，是 context 管理問題，弱模型受害更深。

**修法（照做即可）**：動手前先估——
- 預估要讀 **超過 300 行**、或要開 **超過 3 個檔**、或要做「找找看在哪」式的搜尋 → 派 subagent（Explore 或 general-purpose），主對話只收結論＋`檔案:行號`。
- 已知確切檔案與位置的單點查證 → 自己用 Read/Grep 讀那一段即可，不必派。
- subagent 回報一律遵守回報合約（見 `playbooks/model-dispatch.md`）：只回結論，長產物存檔傳路徑，禁止貼原文大段。

**判準**：回頭看最近 10 則對話內容，若超過一半是檔案原文或工具原始輸出（而非結論），就是下場過深，立刻改派工。

## 第 3 名：自驗自過、先宣告後執行（最易出錯）

**病灶**：兩個變體——
1. 「已完成／已更新」寫在 tool call 成功之前（舊 GLOBAL.md:49 已禁止，但無執行程序，仍反覆發生，見 `_context/lessons-learned.md` 多條）。
2. 做的人自己驗自己：同一個 context 既寫又驗，會沿用同樣的錯誤假設，驗不出自己的盲點。git add pathspec 壞掉整批漏 commit（repo commit 5513dc6）就是自驗漏掉的實例。

**修法（照做即可）**：
- 宣告狀態前先看 tool 回傳。輸出「已完成」四個字之前，逐項對照：這句話對應的 tool call 回傳成功了嗎？沒有就寫「執行中」或「失敗：<原因>」。
- 完成宣告必附證據，格式固定：`驗收項 → 驗證命令/方法 → 實際輸出摘要`。只寫「已確認」不算證據（ody coach 機檢同標準）。
- 檔案類交付：派一個 **fresh-context** subagent read-back（只給檔案路徑與驗收條件，不給寫作過程），回報「存在？完整？符合驗收？」。
- 程式類交付:跑測試或實際執行，貼實際輸出。
- 高風險判斷（會影響使用者決策、或改壞成本高）：加第二意見——換一個模型或換一個 AI（ody 輪替教練制）再判一次。
- 詳細分流見 `playbooks/judgment-rubrics.md`「何時算真的完成」。

**判準**：任何「完成」宣告若拿不出上述格式的證據，就不是完成，是「自認完成」。
