# Auto Approve

啟動本專案本 session 的自動執行模式，直到 PO 指定的下一個驗收節點。

## 觸發方式

`/auto-approve [驗收節點描述]`

範例：`/auto-approve Phase 3 HANDOFF 產出後`

## 執行步驟

### Step 0 — 確認驗收節點（必做）

- 若使用者呼叫時**有提供驗收節點描述** → 記錄，進入 Step 1。
- 若使用者**未提供** → 停止，詢問：「請說明本次自動執行模式的驗收節點（例：Phase 2 完成、地圖頁 ui-review PASS）。」

### Step 1 — 風險評估（必做，不可跳過）

列出接下來預計執行的操作清單，分三類：

| 類型 | 說明 | 範例 |
|------|------|------|
| ✅ 安全 | 唯讀、本地、可還原 | git status、npm run dev、截圖 |
| ⚠️ 低風險 | 修改本地檔案、可 git 還原 | 編輯程式碼、git commit、git push |
| 🔴 需確認 | 不可還原、影響外部系統 | 刪除遠端分支、呼叫第三方 API、rm -rf |

向 PO 報告：「預計執行 X 項操作，其中 N 項需特別注意：[列出 🔴 項目與理由]」

**等待 PO 確認後才進入 Step 2。**

### Step 2 — 寫入權限（PO 確認後才執行）

1. 讀取 `.claude/settings.local.json`（不存在則建立空 JSON）。
2. 在 `permissions.allow` 陣列最前面插入 `"Bash(*)"` 。
3. 若已存在 `"Bash(*)"` 則跳過。
4. 存檔。

### Step 3 — 宣告範圍

回報：
```
✅ 自動執行模式已啟動
範圍：本專案 × 本 session
驗收節點：[PO 提供的描述]
到達驗收節點後，模式自動結束，下次需重新執行 /auto-approve
```

## 重要限制

- 🔴 項目（刪除遠端分支、第三方 API、rm -rf 等）即使在自動模式下，**仍需個別確認**。
- 到達驗收節點後，自動模式視為結束。
- 本 skill 只修改 `.claude/settings.local.json`（專案層級），不影響全域設定。
