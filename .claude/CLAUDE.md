# WTF 專案層設定
> 適用：WTF_Under_Construction 專案。本檔只當索引，不塞長內容

## 專案核心目標

**Workflows That Flow**：以複利方式成長 Claude 協作效率與效益。每次工作都應比上一次更快、更準、更少摩擦。設定、Skills、流程皆為可累積資產，不做一次性修補。

## 開場（本 repo 內工作時）

1. 讀 `wtf-config/GLOBAL.md`（常載核心：開場協議、制度層路由、檔案規範）。
2. 讀 `wtf-config/AGENTS.md`（溝通原則與角色定義正本）。
3. 專案知識走**三檔制**：`_context/INDEX.md` → INDEX 指到的當前 TaskLog → `_context/lessons-learned.md`；嚴禁全量掃描 `_context/`。
   （remote/web session 沒跑過 sync hook 也適用：以上皆為 repo 內相對路徑。）

## 制度層（動手前）

- 派工鐵律與按需路由表在 `wtf-config/GLOBAL.md`「制度層」節。
- playbooks 位置：`wtf-config/playbooks/`（model-dispatch、judgment-rubrics、delegation-templates、maintenance-protocol、harness-diagnosis、letter-from-fable5 等）。
- 改 `wtf-config/` 任何檔之前，先讀 `wtf-config/playbooks/maintenance-protocol.md`。
