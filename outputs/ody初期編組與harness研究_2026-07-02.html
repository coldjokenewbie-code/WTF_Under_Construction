結論先行：Claude Code harness 的四項原生能力（Hooks／Subagents／Headless CLI／持久記憶）足以撐起「靈活應變＋從錯誤學習」的小隊結構。本次已把 Plan_2026-07-01 的三道閘做成可跑的 MVP：契約檢查器＋coach 機檢＋學習規則庫＋Verifier subagent 全數落地並實測（自測 8/8、既有安全測試 30/30、dogfood 契約走完 FAIL→修→PASS 全圈）。Tyrion Stop hook 本日實戰攔截驗證通過（真的擋下一則違規回覆並強制重寫）。

1｜設計精神：Odysseus × Tyrion

  • Odysseus（多謀善變）：不預先寫死流程，任意任務都走同一套「契約→執行→驗收」結構；手段可換（本地腳本／LLM／跨 AI），結構不變。
  • Tyrion（識人守分寸）:「我喝酒，而且我知道事情」——知道自己會漂移，所以不信任自律，把分寸交給機器守門；每次犯錯付「一條規則」的學費，同錯不二犯。
  • 核心命題（沿 Plan 定調）：治理 agent 本身的 harness，靠結構強制，不靠 AI 自覺。

4｜三道閘（本次實作核心）

# 閘1 立契約（無契約不動工）
python3 tools/ody/squad/coach.py new <task_id> --goal "..." \
    --scope "tools/x/*" --accept "標準1" "標準2" --verify-cmd "測試命令" --permission "授權點"

# 執行中：逐條填自驗證據（cmd 會真的執行並記 exit/輸出尾段）
python3 tools/ody/squad/coach.py evidence <task_id> 1 --cmd "python3 .../test.py"

# 閘2 宣稱完成 → 機檢驗收
python3 tools/ody/squad/coach.py check <task_id>   # PASS / FAIL＋逐項原因

# 閘3 FAIL/糾正 → Mentor 轉規則（下次 check 自動全套用）
python3 tools/ody/squad/coach.py add-rule --rule-id R00x --type banned_path_in_diff ...

閘2 檢查項：契約完整／scope 越界（git diff＋dirty 逐檔比對 allowlist；開工前既髒檔豁免）／自驗證據逐條（cmd 須 exit 0、note 禁空話）／handoff ≤60 行／verify_cmds 全過／coach_rules 全套用。事件全部入 events.jsonl（schema 驗證，供 learn.py 與回顧）。

7｜誠實揭露（限制與違規記錄）

  • 本任務自己違反了閘1：coach.py 與 schema 是「先動工、後立約」——正好示範沒有機器強制點就會違規。事後已把這兩檔從既髒豁免名單移回受檢範圍；根治＝PreToolUse 契約閘（下一步）。
  • Verifier subagent 仍是同一個模型（context 分離、非廠牌分離）；Plan 的「換不同 AI」要靠 headless CLI 接線後才完整。
  • subagent 的 Bash 無法硬性唯讀（靠指令約束）；硬限制同樣要 PreToolUse。
  • 「立約前動工」「結論先行」這類時序/語意規範目前無法全自動機檢，列 checklist 人工核。

8｜下一步（建議優先序）

  1. PreToolUse 契約閘（opt-in）：無契約擋 Write/Edit——把閘1從紀律變強制。
  2. 跨 AI coach 接線：coach.py 加 --peer codex|agy，機檢後的判斷項交不同廠牌複核（輪替防放水）。
  3. 驗穩推全域：Tyrion 規則進 CODEX.md/GEMINI.md＋~/.claude/settings.json（需 PO 授權）。
  4. 引文白名單（修 Stop hook 誤攔）；/ody skill 一鍵入口；SessionEnd 聚合違規供 nightly 回顧。
