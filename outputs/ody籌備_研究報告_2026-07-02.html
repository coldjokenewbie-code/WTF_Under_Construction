🏹 奧德賽小隊（ody）籌備研究報告

2026-07-02｜ai-team 三方討論收斂｜精神：Odysseus 多謀善變＋從經驗學／Tyrion 謀略守分寸
框架已改名 assistant → ody基本功優先

1. 根因：為何規範不落地、無複利

規範停在 prompt 層，沒有輸出前檢查與違規阻斷。LLM 的禮貌/完整/解釋慣性會蓋過「極簡」規則 → 靠自律必漂移。實測：Claude 一再違反「極簡/禁聊天語氣」，需使用者重複提醒、退回原點 = 無法複利學習。三方一致：唯一解是結構強制，不是更多提醒。

2. 解方：輸出守門 + 錯誤轉規則

Tyrion 守門：交付前用 reply_lint.py 掃描 → 命中禁詞（好的/當然/讓我/完美/成功為你…）即 block 退回重寫；過長只警示。
Mentor 學習：每次糾正/錯誤 → 轉成 lint_rules.json 一條可機檢規則（禁詞/門檻）→ 下次自動擋。學習＝新增規則，不是寫反省。
複利＝越用規則庫越大、同類錯誤再也過不了關。跨工具 hook 不對等（只 Claude Code 有原生 Stop hook）→ 強制點放外部 lint 腳本，三工具皆可用。

3. 編組（4 角色，動態擔任）

角色職責
Odysseus策略/執行：拆任務、判風險、決定誰做、增量實作
Tyrion守門：交付前 lint（禁詞/過長/結論先行），違規退回重寫
Mentor學習：把錯誤轉成 lint_rules.json 一條可機檢規則
Verifier驗收：對照需求/diff/測試/截圖；禁自驗自過（換 agent）

4. 最小可落實規則（今日）

① 結論先行、禁聊天語氣、散文 ≤500 字（Tyrion 硬擋禁詞）｜② 未要求不主動讀寫/改碼/開計畫｜③ 犯錯即停工→先寫一條規則→再續｜④ 授權點開頭一次講定、過程不中途要權限｜⑤ 視覺分只看實圖（禁盲評當數字）｜⑥ 禁自驗自過

5. 自評 + 學習迴圈

違規/錯誤 → Mentor 轉可機檢規則寫入 lint_rules.json → Tyrion 下次自動套用 → 同類錯誤再也過不了關（不再犯 = 複利）

6. 落地狀態

元件狀態
reply_lint.py 守門引擎✓ 已建、測過（禁詞抓到、精簡放行）
lint_rules.json 規則庫✓ 初版（禁詞表＋字數門檻）
stop_hook.py 交付前自動 lint✓ 已建、測過（block/放行/防迴圈/fail-open）
