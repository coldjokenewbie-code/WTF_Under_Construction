# ody 小隊 PRD（奧德賽小隊組建需求文件）
> 讀者：AI（預設 Claude Code，你將擔任 Tech Lead 兼籌備負責人）
> PO：使用者本人。本文件定義「要什麼、為什麼」；「怎麼做」由你研究自身環境後提案。
> 版本：v1.0（2026-07-03，源自 WTF 專案實戰驗證的架構）

---

## 1. 問題：為何要組小隊

單一 AI 有三個**結構性缺陷**，靠 prompt 提醒無法根治：

1. **自驗自過**：自己寫、自己驗，同一個 context 看不見自己的盲點。「宣稱完成」經常不等於「真完成」。
2. **角色利益衝突**：執行時求快、驗收時該挑剔、守門時該嚴格——一個腦袋切換不乾淨，總會自我妥協。
3. **學習不留存**：session 結束教訓就丟，同樣的錯下次照犯。寫在 prompt 裡的規範會隨對話變長而漂移（自律必漂移）。

**解法**：把一個 AI 拆成多個「獨立 context 的角色」互相制衡，並用機器強制（harness）取代自律。

## 2. 好處（含誠實標註）

### 品質面
1. **杜絕自驗自過（結構性）**：Verifier 用獨立 context 複驗，不靠自覺。原型實測：dogfood 兩份契約抓出 3 個執行者自己沒發現的真 bug。
2. **驗收視角增加**：不同模型輪流驗收，訓練偏性不同，A 的盲點 B 看得見。
3. **角色利益分離**：執行／守門／驗收／學習分給不同成員，避免自我妥協。

### 風險面
4. **不單壓一個模型**：各模型都有突破期與降智期（版本更新、服務品質波動）。小隊架構下 Lead 可換人、執行層可替補，單一模型出狀況不癱瘓工作流。
5. **學習資產不綁供應商**：規則庫、契約、教訓全是 repo 內的純文字檔，模型升級換代照用。

### 效率面
6. **最少化人工介入**：headless 直驅取代人工轉貼＋前置授權一次談定＋機檢取代人眼中判＋協議預寫收斂條件。用戶只剩兩個接觸點：**開頭定需求授權、結尾驗收成品**，不必分散注意力時時監督。
7. **可平行**：執行層多成員可同時開工（git worktree 隔離），不互相等待。

### 成長面
8. **學習共享、期待複利**：錯誤轉成可機檢規則入共用規則庫，全隊、跨專案生效，同錯不再犯。**誠實標註：複利效果目前尚不明顯**——已見效的是單點回本（規則建立當日即攔到真錯誤），複利需時間累積驗證。請如實向 PO 管理預期。

## 3. 編組：如何組成小隊

四個角色。**Phase 1 全部用「同一個 CLI 的不同機制」實現**，不需要第二個 AI 工具：

| 角色 | 職責 | Phase 1 落地件（Claude Code） |
|---|---|---|
| **Odysseus**（執行） | 立契約、執行任務、填證據 | 主對話本身 |
| **Tyrion**（輸出守門） | 攔截違規輸出（禁詞、冗贅），強制重寫 | Stop hook ＋ lint 腳本＋規則檔 |
| **Verifier**（獨立驗收） | 對「宣稱完成」逐條複驗、重跑驗證命令，回報 PASS/FAIL | subagent（`~/.claude/agents/*.md`，天然獨立 context） |
| **Mentor**（學習） | 把 FAIL／糾正轉成可機檢規則 | 規則庫 JSON ＋ add-rule CLI |

**關鍵原則：學習＝加一條機器可檢查的規則，不是寫一段反省文。** 反省文會漂移，規則會執行。

## 4. 協作通道：最少化人工介入的設置

以前的做法是用戶當中介、在多個 AI 之間人工轉貼對話。本架構用四件事取代：

1. **headless CLI 直驅**（Phase 2）：主流 AI CLI 都有無頭模式（`claude -p "…"`／`codex exec "…"`／`gemini --print "…"` 等）。Tech Lead 直接用 Bash 呼叫其他 agent、stdout 即時拿回回應——人工轉貼變成一行 subprocess。注意：headless 每次呼叫都是無狀態新 session，**多輪討論由 Lead 每輪彙整前情餵進 prompt**（Lead 就是自動化的中介人）。
2. **星狀拓撲**：不是 N 個 agent 互聊（那才需要中介），而是一個 Lead 當 hub，其他成員被程式化呼叫。誰當 Lead 都對稱可行，預設 Claude。
3. **共用檔案系統當留言板**：契約、討論紀錄、驗收結果都寫在 repo 內共用檔；agent 間傳大內容靠讀寫同一份檔，不塞 prompt。
4. **把「人的判斷點」換成機器判斷＋預寫協議**：
   - 中途不問「做完了嗎」→ 機檢（exit code）＋ Verifier 複驗替代人眼
   - 中途不問「可以嗎」→ **前置授權**：開工前一次列出所有需授權項讓 PO 談定
   - 循環自行收斂 → 協議預先寫死（例：動工前至少討論、輪流驗收、驗收上限 N 輪、收斂不了才交回 PO）

## 5. harness：成隊後的紀律（三道閘）

harness ＝ 包在模型外、以程式強制執行的約束層。三層強度：模型（機率）＜ prompt（自律、會漂移）＜ harness（程式強制、繞不過）。小隊規範必須落在 harness 層。

- **閘 1｜動工前立契約**：任務開始先產生契約檔（goal、scope 白名單、驗收條款、驗證命令、授權點）。無契約不動工。
- **閘 2｜完工前機檢**：宣稱完成前跑檢查器——scope 有無越界（git diff 比對白名單）、每條驗收有無真實證據（命令 exit code，禁空話如「已確認」）、規則庫全套用。FAIL 就修，不准帶 FAIL 交付。
- **閘 3｜FAIL 轉規則**：每個 FAIL 與 PO 糾正，收斂成一條可機檢規則進規則庫，下次自動攔。

**雙學習迴圈**：輸出層（PO 對回覆風格的糾正 → lint 規則 → Stop hook 攔）＋工作層（任務 FAIL → coach 規則 → 機檢攔）。

## 6. 分期與驗收標準（DoD）

### Phase 1 — 單 CLI MVP（先做這個）
交付物：
1. Stop hook ＋ lint 腳本＋規則檔（Tyrion）
2. 契約檢查器 CLI：new／evidence／check／add-rule 四個子命令（閘 1/2/3）
3. Verifier subagent 定義檔
4. 規則庫初始 2 條（保護路徑需明授、證據禁空話）
5. 研究報告給 PO：你的環境 harness 能力對照表＋落地件清單

**DoD：拿一個真任務 dogfood 走完「立契約 → 執行 → 機檢 FAIL → 修 → PASS → Verifier 獨立複驗」全程，並誠實記錄過程中自己違規的地方。**

### Phase 2 — 跨 agent（可選，裝了第二個 AI CLI 再做）
- headless 互驗：驗收命令改派另一家模型跑（跨模型視角）
- Lead 直驅派工與多輪討論

### Phase 3 — 強化
- PreToolUse hook：無契約直接擋 Write/Edit（閘 1 的機器強制，根治「先動工後立約」）
- 誤攔白名單（引文／測試脈絡）

## 7. 已知坑（原型實戰踩過，直接避開）

1. **hooks 不熱載**：改 settings 裡的 hook 要新 session 才生效；**agents 與 skills 熱載**（部署即生效）。驗證 hook 務必開新 session 測。
2. **hook 必須 fail-open ＋防迴圈**：lint 腳本掛掉時要放行（不可鎖死對話）；Stop hook 要檢查 `stop_hook_active` 旗標防止無限攔截循環。
3. **git 機器解析兩坑**：非 ASCII 檔名預設被八進位跳脫（加 `-c core.quotepath=false`）；`status --porcelain` 首行前導空白若被 strip 會使欄位錯位、路徑掉首字（逐行解析、不要先 strip 整段）。
4. **git add 壞 pathspec 整批中止**：一個 pathspec 失效會使整個 add 批次靜默失敗，commit 漏檔。不要 `2>/dev/null` 吞錯，commit 後核對 files-changed 數。
5. **守門誤攔引文**：回覆中「引用」禁詞（如展示 lint 測試輸出）會被自己的 hook 攔。Phase 1 先接受、記錄，Phase 3 做白名單。
6. **headless 無狀態**：跨 agent 多輪討論，Lead 必須每輪重餵脈絡，別假設對方記得上一輪。
7. **紀律必漂移的實證**：原型開發時執行者自己就違反了閘 1（先寫程式後立契約）——這正是要做 PreToolUse 機器強制的原因。預期你也會犯，犯了就誠實記錄。

## 8. 邊界與授權（鐵則）

- **前置授權**：開工前一次列出所有需授權項（要改的全域設定、要裝的東西、要花錢的 API）讓 PO 談定；中途不得追加索權。
- **保護路徑**：全域設定檔（`~/.claude/settings.json`、共用規範檔等）非明授不得修改——這本身就是規則庫第一條。
- **禁自行使用付費 API**。
- **不自行 commit**：完成後交 PO 驗收，PO 點頭才 commit。
- **誠實回報**：FAIL 就說 FAIL，帶原始輸出；不得美化。
