結論（建議工作流，三層）

  1. 主力＝現有 GCP Vertex AI（零建置，今天就能商用）：量產圖走 Imagen 4 Fast $0.02/張 或 Gemini 影像系 $0.034–0.067/張；影片走 Veo 3.1 Lite $0.03/秒 起、關鍵產出用 Veo 3.1（含音訊 $0.40/秒）。全線經官方定價頁驗證，且是本次比較中唯一有官方 IP 侵權賠償（indemnification）的方案。
  2. 外掛＝Ideogram API（圖內文字／海報排版場景）：按張付費 $0.03–0.10、無月費；文字渲染是它的護城河。免費網頁層不可商用（保守解），影片功能不存在。
  3. 第二階段＝ComfyUI＋雲端 GPU（量大或要客製控制時再投入）：FLUX.1 schnell、Qwen-Image、Wan 2.1/2.2 全部 Apache 2.0 免費商用（LICENSE 已驗證），雲端 4090 約 $0.17–0.44/hr，推算單價比 API 低一個量級（圖 ~$0.002/張、5 秒影片 ~$0.02–0.05）。代價＝workflow 學習與維運、無 IP 賠償；MacBook Air 只能當學習機。

避開：一切「免費層商用」（Ideogram／Kling／Runway／Hailuo 免費層皆禁或不建議）；Sora 2 API（2026-09-24 日落）；FLUX.1 dev 權重自架商用（灰區）；Hunyuan 系（硬體門檻＋排除歐盟/英國/南韓地區條款）。

2. Ideogram：何時值得用

  • 用它的理由只有一個：圖內文字。搜尋摘要稱其文字正確率 ~90–95%（對照 Midjourney ~30–40%）。海報、Logo 字樣、標語圖、簡報封面等「字要對」的場景，它仍是首選。
  • 最低成本商用路徑＝API 按張計費（Turbo $0.03/張），比訂閱 $20/月划算——除非月產量大且偏好網頁介面。
  • 兩個陷阱：(1) 免費網頁層——Licensing 類來源稱僅限非商用、產出強制公開，ToS 摘要卻稱不限制商用；兩說矛盾，決策採保守解：免費層不商用，待人工開 ToS 核實。(2) 4.0 雖釋出開放權重，授權是 Non-Commercial Model Agreement——自架照樣不能商用。
  • 無影片功能（多輪多來源一致）。影片需求由 Veo／Wan 補。

5. 融入現有工作（WTF）

  1. 已有資產：gen-tools/ 三支 Vertex 腳本（Imagen／Nano Banana／Veo）＋已開 billing 的 GCP 專案（RESOURCES.md 登記）。
  2. 提案（待核准後改 RESOURCES.md）：腳本預設模型升級——imagen-3.0-generate-002 → Imagen 4 系（更便宜且新）、veo-3.0-generate-001 → Veo 3.1 系（Lite/Fast 檔位新增）；並注意 Imagen 4 有 8/17 棄用傳聞（未經官方頁確認）——新流程優先掛 Gemini 影像系減少遷移次數。
  3. 要用 Ideogram 時：建 gen-tools/ideogram_generate.py（API 按張），只在「圖內文字」場景呼叫；其餘圖像一律走 Vertex（省錢＋有賠償）。
  4. ComfyUI 起步（可選）：MacBook Air 先裝來學 workflow（SDXL）；確認 Windows 桌機 GPU 後再決定本地或 RunPod。第一個目標 workflow 建議＝Wan 2.1 1.3B 圖生影片（8GB 門檻最低、Apache 2.0）。

7. 研究方法與限制
