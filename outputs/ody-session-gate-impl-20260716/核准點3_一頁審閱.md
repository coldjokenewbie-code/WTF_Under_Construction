# 核准點 3：要不要讓「保險絲」正式上線？

## 你在核准什麼（一句話）

允許我修改你電腦上的兩個 Claude Code 設定檔，讓「模型沒載入全域設定就不能工作」從提案變成現實。

## 目的

解決這件事：模型開場經常沒讀 GLOBAL/AGENTS 就開工（例：上次 Studio 開錯資料夾）。文字提醒已證明擋不住，所以改用機器攔截。

## 上線後會發生什麼

1. 每次開新對話，Claude Code 會**自動**把 GLOBAL.md＋AGENTS.md 全文放進模型腦中（用官方載入機制，不經過模型意願）。
2. 載入成功會留下「收據」。**沒有收據＝模型的所有工具被鎖死、也不能正常結束回合**——它不讀就什麼都做不了。
3. 你從此不用再問「你讀了沒」。

已在隔離環境實測：會載入、會鎖、鎖得死（模型嘗試 9 次都出不去）。

## 效益

- 「開場沒讀設定」這類事故從「靠模型自覺」變成「機器保證」。
- 每次開對話多花約 11KB 上下文（全域設定本身的大小，先前瘦身過）。

## 風險（誠實版）

| 風險 | 說明 | 保險 |
|---|---|---|
| 設定改壞，Claude Code 開不了工 | 改的是 `settings.json`（掛鉤）＋`CLAUDE.md`（加 2 行） | 改前備份；出事一行指令還原 |
| 誤鎖：載入慢半拍，開場前幾秒工具被暫時擋 | 實測未發生，但理論上可能 | 只在這台機器先跑，觀察幾天再擴 |
| 有一個未知數：正式環境的自動載入可能跳出一次「核准視窗」要你點同意 | 上線第一步就是開一次互動視窗確認 | 若行為不符預期，立即停止並還原 |
| 這不是防駭安全機制 | 有意繞過的人（或模型用特殊旗標啟動）仍可關掉它 | 它管「忘記讀」，不管「蓄意繞」 |

## 你的選項

- **核准** → 我：備份 → 改兩檔 → 開互動視窗驗證 → 實測一輪 → 回報。全程可還原。
- **暫緩** → 一切維持現狀，已完成的程式與測試躺在 repo 裡不生效。
