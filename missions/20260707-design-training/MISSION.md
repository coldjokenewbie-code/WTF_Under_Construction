# MISSION：design-training（使用者設計能力訓練,常設）
## 方向
依同目錄 `訓練綱領.md` 持續運作使用者的視覺/3D 訓練支援。綱領是裁決,不得改動;本 mission 無終點,以週為循環。

## 弱模型職責（每週循環的 backlog 模板）
1. **案例包**：選一案（優先：世界級展場/博物館導覽/工業遺產展示），依綱領三問寫分析,附**實開驗證過的來源連結**（未實開禁列）。存 `out/case-YYYYWW.md`。
2. **練習出題**：依 A/B/C 輪替與使用者近期專案出三題（題目具體到「畫什麼、幾分鐘、進哪個交付物」）。寫入案例包尾。
3. **批評官值勤**：使用者上傳照片時依綱領回饋迴路第 1 條執行（一次一事、並列比較、禁讚美）。**基線與練習照片位置**（Mac 本機 Drive，雲端不可及→批評值勤只能由本機 session 執行）：`~/Library/CloudStorage/GoogleDrive-coldjokenewbie@gmail.com/其他電腦/tachart_ihuy/Claude_cowork/projects/Aseembly_Plant_Interactive_machine/Baseline/`。基線建立後摘要與錨點寫回 WTF `wtf-config/anchors/手繪.md`（隨 git 跨機）。
4. **月審**：每四週把錨點鏈並列成一頁 HTML（`out/progress-YYYYMM.html`），標進步/停滯,停滯項依綱領停損條處理。

- 出題與使用者當前專案掛鉤（查各專案 INDEX 近況）,禁通用美術題。
