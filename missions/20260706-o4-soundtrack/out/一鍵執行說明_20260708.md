# O4 配樂重做 v4：本機一鍵執行說明（路線 C × 方向二）

> 前提：使用者已在 `_blockers.md` 的討論閘拍板「方向二：制度的脈搏」。**拍板前不要跑 `generate_stems_20260708.sh`**（會產生計費的音檔生成）；`assemble_v4_20260708.sh` 本身不呼叫外部 API，但沒有 raw 素材也跑不動。

## 步驟

1. **確認環境**：本機需要 `gcloud`（已登入 ADC：`gcloud auth application-default login`）、`ffmpeg`（`brew install ffmpeg`）。
2. **生成 6 段素材**（呼叫 Vertex AI lyria-002，計費）：
   ```bash
   cd claude_CDIC_O4/outputs/配樂重做_20260708
   GCP_PROJECT=<你的 GCP project id> GCP_LOCATION=us-central1 ./generate_stems_20260708.sh
   ```
   產出 `raw/opening_S1.wav` ~ `raw/opening_S4.wav`、`raw/event1985_E1.wav`、`raw/event1985_E2.wav`（各 30s）。
   若某段失敗或欄位不符，腳本會保留 `raw/<段名>.response.json`，開檔核對 Vertex 實際回傳欄位再回報。
3. **接合**（純本機運算，不計費）：
   ```bash
   ./assemble_v4_20260708.sh
   ```
   產出 `Opening_v4.mp3`（66.3s）、`Event1985_v4.mp3`（40.0s），並印出實測長度供核對（容許 ±0.05s）。
4. **實聽驗收**（對照 `../配樂定調書_2026-07-08.md` 對齊驗收表）：
   - 26.0s／48.6s／60.5s（Opening）與 23.25s（Event1985）三＋一個接合處，聽有沒有明顯拼接感。
   - 13.6–17.5s 是否撐得住「1929 大蕭條」的情緒重量。
   - 39.4s FDIC 是否有聽得出的「明亮和聲」轉折。
   - 65.3s 起是否平順收束到近似寂靜。
5. **驗收過 → 回報使用者**：接線（`public/music/` 複製＋`Root.tsx` `music_version=4`）是下一個 backlog 項，待這輪實聽通過後才做，本次不動。若聽感不理想，把哪一段/哪個接合點的問題寫回 `../_blockers.md`，不要自行重跑生成（會產生額外計費）。

## 已知限制（誠實揭露，不假裝查證過）
- lyria-002 回傳 JSON 的音訊欄位名稱（`bytesBase64Encoded`）沿用 Vertex 其他生成媒體 API 慣例，**未針對 lyria-002 逐字核對官方文件**（本次網路白名單擋掉 `docs.cloud.google.com`，見 `../music-research_2026-07-08.md` 開放問題1）。若欄位不符，`generate_stems_20260708.sh` 會印出錯誤並保留原始回應供核對。
- 接合點/用量秒數已用公式反推核對過（見 `prompts_route-C_direction2_20260708.md` 末節），但實際生成內容仍是 AI 生成、非決定性——同一 prompt 每次生成的音色細節會有變化,請以實聽為準,不是只信任算式。
