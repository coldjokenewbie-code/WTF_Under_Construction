# MISSION：o4-soundtrack（存保 O4 開場影片配樂）
## 方向（一句話）
為 claude_CDIC_O4 開場影片產出**世界級展覽等級的配樂方案**：設計書＋逐段時間碼＋可直接生成的 prompt 包；音檔生成留本機（雲端無 GCP 憑證，推測待證）。

## 品質定位
- 世界級的操作定義：配樂結構逐秒對齊畫面敘事（時間碼精確到 0.5s）、情緒曲線有論證（為何此處收、此處放）、參考曲例可查證（遵「參考資料必先驗證」規則，未實聽/實查的連結禁列）。
- 題材：1980s 台灣金融記憶→存保制度——懷舊不濫情，機構感不冰冷。禁罐頭企業配樂語彙（上揚弦樂+鋼琴琶音的預設組合須有理由才可用）。

## 素材（唯讀）
repo `claude_CDIC_O4`：`public/videos/opening_2.mp4`、`opening_8.mp4`（現行版）、PRD.md、O區素材清單、information/。**repo 不可及→blocker 後結束本棒，禁 parked。**

## 產出位置
WTF `missions/20260706-o4-soundtrack/out/`：
1. `配樂設計書.html`——段落情緒地圖（畫面敘事 vs 音樂事件對照表，含時間碼）、配器與參考語彙、兩版方案（懷舊敘事型／節制機構型）。
2. `prompts.md`——每段雙格式生成 prompt（Vertex Lyria／Suno），含 BPM、調性、時長、銜接說明。
3. `generate_local.sh` stub——本機一鍵生成的腳本骨架（呼叫哪個 API、參數位、輸出檔名規則）。
4. 對齊驗收表——生成後逐段核對的 checklist（時間碼、轉場點、結尾收束）。

## 硬底線
1. 影片實際時長以 repo 內檔案量測為準（能跑 ffprobe 就跑，不能就用 HTML5 video 讀 duration），禁目測。
2. 時間碼表覆蓋 100% 片長，無空窗。
3. 每個參考曲例附「實際查證方式」；查不到的寫查不到。
4. 兩版方案都完整可生成，供使用者挑選（挑選結果依 anchors/README 回寫 `anchors/配樂.md` 首批錨點）。

## Milestone
1. 時長量測＋畫面敘事分段＋情緒地圖＋方案 A 完整（設計+時間碼+prompt）。
2. 方案 B 完整＋生成腳本＋驗收表＋對抗審查 → done。
