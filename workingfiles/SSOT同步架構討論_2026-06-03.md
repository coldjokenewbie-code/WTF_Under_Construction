> 目的：讓「開啟任意專案、任意 AI 對話」都能自動讀到最新的全域設定／skills／知識管理／工作規範／檔案規範。

> 收斂後在末段共同寫「結論」。

## Windows 端現狀

- Antigravity 的 Gemini／其他非 Claude 工具：靠讀取專案根的 `AGENTS.md`／`GEMINI.md`，但**不會自動跑 sync_config.py**。

## 問題盤點（Windows 視角）

1. **觸發是手動的**：sync 要人/AI 主動跑。換機、換專案、SSOT 更新後，若沒跑 sync，各專案副本就過期。
2. **跨工具不對等**：只有 Claude Code 能掛 hook 自動跑；Gemini/Codex 只能被動讀已存在的副本，無法自我刷新。
3. **雙重同步管道的時序風險**：SSOT 在 GitHub repo 內，但被 Drive 專案消費。要讓 Drive 專案拿到最新 SSOT，順序必須是 **git pull WTF repo → 再跑 sync_config.py**。只靠 Drive 同步，WTF repo 內容可能落後（git 與 Drive 對 .git 的處理也可能衝突）。
4. **「任意 AI 自動讀最新」目前做不到全自動**：最多做到「Claude Code 用 hook 自動 sync」＋「其他工具讀到上次 sync 的副本」。

**P1. 確立單一觸發點：SessionStart 自動 sync（Claude Code 兩端都掛）**
- 在 `~/.claude/settings.json` 掛 SessionStart hook：`git pull`(WTF repo) → `python wtf-config/sync_config.py sync`。
- 效果：每次開 Claude Code session 自動把最新 SSOT 推進所有專案 + ~/.claude/。其他工具隨後讀到的副本即為最新。

- 建議把 `wtf-config/` 的權威同步交給 git（pull/push），避免 Drive 對 .git 的雙重同步衝突。
- Drive 仍同步「各專案產出的副本與工作檔」，但 wtf-config 的更新一律走 git。

**P3. 跨工具靠「專案根副本」對齊（已有，維持）**

**P4. 容錯：把 `check` 結果攤給用戶**
- hook 跑 sync 後，session 開場由 AI 報告 check 摘要（幾個 STALE/BROKEN 已修）。符合 GLOBAL.md「AI 衰退因應：關鍵狀態寫檔、不靠口述」。
