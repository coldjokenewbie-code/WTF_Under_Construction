## PRD v1

**環境**：MS365 Apps 商務版，10–50 人

### 資料來源（皆在 SharePoint）

| 資料 | 來源 |
|---|---|
| 出勤歷史 | SharePoint List `AttendanceHistory` |
| 請假申請 | Excel（假別 / 日期 / 狀態 / 員工 Email） |
| 假期餘額 | Excel（假別餘額 / 員工 Email） |

### AttendanceHistory 欄位
日期 / Email / 員工編號 / 姓名 / 方案名稱 / 任務名稱 / 表單回覆 / 實際出勤狀態 / DateString

### 出勤查核背景流程（已完成，非開發範圍）
每日 09:30 Power Automate 查 Planner 截止日已過工作 → 寄信行政 → Line 通知當事人 → 行政現場核對 → 手動寫入寬格式 Excel → 每日中午 Office Scripts 轉長格式寫入 `AttendanceHistory`

### 功能需求

**模組一：出勤查核（開發中）**
- 顯示當天有延遲工作的紀錄（Filter by Email + 今天）
- 員工填寫「表單回覆」：下拉（已處理 / 待確認 / 其他）+ 選「其他」顯示文字輸入框
- Patch 回 `AttendanceHistory.表單回覆`
- 實際出勤狀態欄唯讀（行政填入）

**模組二：請假紀錄（待開發）**
- 顯示員工本人請假紀錄（假別 / 日期 / 狀態）、月份切換

**模組三：假期餘額（待開發）**
- 顯示各假別剩餘天數

### 角色權限

| 角色 | 判斷方式 | 權限 |
|---|---|---|
| 一般員工 | 預設 | 填寫表單回覆、查看自己紀錄 |
| 行政人員 | 特定 Email | 另可編輯實際出勤狀態 |

### 技術設定
- 識別員工：`User().Email`
- 委派查詢：DateString 文字比對（避免 `Year()` / `Month()` 委派限制）
- 響應式：關閉「縮放以符合視窗大小」，全用 `Parent.Width` / `Parent.Height` 比例
- 嵌入：SharePoint 頁面，供瀏覽器存取

### 開發順序
1. 模組一 Popup 修正（進行中）
2. 模組二 請假紀錄
3. 模組三 假期餘額
4. 行政角色權限
5. 嵌入 SharePoint

## 累積 Lessons

**[2026-04-09] Popup 置中唯一可靠做法**
Power Apps AutoLayout 容器內 X/Y 無效，Popup 置中唯一可靠做法是將 OverlayContainer 放在 Screen 根層級（與主容器同層），再用 LayoutJustifyContent.Center 置中。
來源：PopupContainer 放在 ScreenContainer1 內導致位置失效，移出後解決。

**[2026-04-10] Popup 必須放在樹狀檢視最上方**
Power Apps 樹狀檢視中元件越下方圖層越底部，Popup 必須放在 Screen1 的第一個子元件才會蓋在其他內容上方。
來源：PopupContainer 放在 ScreenContainer1 下方，被壓在最底層看不到。

**[2026-04-10] Classic/DropDown 取值用 Selected.Value**
`Classic/DropDown` 取選取值用 `Selected.Value`（字串）；`SelectedText` 是 Record 型別，用於比較或 Patch 會報型別不符錯誤。
來源：Patch 表單回覆時用 `SelectedText` 導致「比較不相容的類型」錯誤。

**[2026-04-10] Dropdown 自由輸入的 Default 處理**
Dropdown 有自由輸入情境時，`Default` 需搭配 `in` 判斷既有值是否在選項內，不在則預設「其他」並將既有值填入 TextInput，避免自由輸入值遺失。
來源：既有表單回覆為「需要溝通」時，重新開啟 Popup Dropdown 顯示「已處理」而非「其他」。
